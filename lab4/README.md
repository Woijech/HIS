# Лабораторная работа №4
## Хеш-таблица с разрешением коллизий через цепочки: сбалансированное дерево

Проект реализован на **Python 3** и использует только **стандартную библиотеку**.

## Что реализовано

- CRUD-операции над хеш-таблицей: добавление, поиск, обновление и удаление;
- разрешение коллизий через цепочки, где каждая корзина хранится как **AVL-дерево**;
- вычисление числового значения ключа `V(K)` по **первым двум буквам**;
- вычисление адреса по формуле из методички: `h(V) = V mod H + B`;
- контроль дубликатов ключей;
- отображение содержимого корзин, вычисленных `V(K)` и `h(V)`;
- расчёт коэффициента заполнения таблицы и статистики коллизий;
- демонстрационный набор данных на 12 записей с несколькими коллизиями.

## Формула хеширования

Используется русский алфавит из 33 букв:

```text
А=0, Б=1, ..., Я=32
```

Числовое значение ключа вычисляется по первым двум буквам:

```text
V(K) = p1 * 33 + p2
```

где `p1` и `p2` — номера первых двух букв ключа.

Хеш-адрес вычисляется так:

```text
h(V) = V mod H + B
```

где:
- `H` — размер таблицы;
- `B` — базовый адрес.

## Архитектура

Проект разделён на слои:

- `hashlab/domain` — доменные модели, AVL-дерево, хеш-функции и сама хеш-таблица;
- `hashlab/application` — сервис сценариев использования и демо-данные;
- `hashlab/presentation` — консольный интерфейс и форматирование вывода.

Такое разделение позволяет соблюдать принципы **SOLID** и упрощает тестирование.
